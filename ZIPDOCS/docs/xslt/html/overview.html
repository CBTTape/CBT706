Overview of the chart layout

Chart Layout

The chart is divided into five main areas (North, South, East, West and the Plot Area) as follows:

The main area is the plot area. This is where the line chart is drawn. The size of the plot area is computed from the information you specify in the <layout> section of the XML input file. You start by specifying the width and height of the entire chart. As you specify the widths or heights for the other areas, those areas will be subtracted from the available space and what is left over is used as the plot area.

The syntax on the diagram above is called XPath. It is a shorthand way of locating a particular node in an XML document. For example, the '/chart/layout/width' refers to the <width> element under the <layout> element under the <chart> element.

Notes about layout

  • You can see where all the areas have fallen into place by specifying the debug="true" attribute on the <chart> element. This will help you adjust the sizes of each area.
  • Although width and height can both be specified for all areas, only the <height> elements are used for the North and South areas, and only the <width> elements are used for the East and West areas. This is because the other dimension is computed in each case.
  • Within each area you can specify how high (in pixels) the text is to be by specifying the <fontsize> element. This is used to center the text inside that area. To confuse things, you can also specify the CSS font-size style to indicate the size of the text to use, but this will not be recognized by the centering computation and you could end up with something not out of place in a Picasso exhibition.
  • With each label area (<label> element) you can specify that the labels be rotated by specifying a <rotation> element. Rotations are in degrees and a positive value rotates the text clockwise.
  • The North title and label areas (used for the secondary X-axis) are not normally wanted, so they will only appear if you explicitly code a North <label> element. For example: <chart><layout><north><label/>...
  • You can only get a legend to appear in the East and/or South areas. Only truly twisted people put legends in the North and West areas. This restriction is for their own good.
  • You can specify the number of columns of legend entries to appear by specifying the <columns> element under the <legend> element.
  • The inset value is internally fixed at 5 pixels. You would not normally need to change it. I never have.
  • The chart margin value defaults to 10 pixels, but you can change it by specifying the <margin> element.

Sample <layout> Element


<chart>
   .
   .
    <layout>
        <width>800</width>
        <height>600</height>
        <margin>10</margin>
        <title>
            <height>40</height>
        </title>
        <north>
            <title>
                <height>20</height>
            </title>
            <label>
                <height>10</height>
            </label>
        </north>
        <south>
            <label>
                <height>10</height>
                <fontsize>10</fontsize>
                <rotate>-90</rotate>
            </label>
            <title>
                <height>20</height>
            </title>
            <legend>
                <height>60</height>
                <columns>4</columns>
            </legend>
        </south>
        <east>
            <label>
                <width>50</width>
            </label>
            <title>
                <width>20</width>
            </title>
            <legend>
                <columns>1</columns>
            </legend>
        </east>
        <west>
            <title>
                <height>20</height>
            </title>
            <label>
                <width>50</width>
                <height>10</height>
            </label>
        </west>
    </layout>
   .
   .
</chart>
