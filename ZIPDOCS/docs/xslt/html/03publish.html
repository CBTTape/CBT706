How to publish SVG using Apache

  • You may need to configure your web server to associate svg (and svgz) files with the "image/svg+xml" MIME type. This lets the web server tell the web browser that the file being sent to it should be rendered by something that understands the SVG format (for example, the Adobe SVG plugin). Either update your httpd.conf file or add the following to .htaccess in the directory in which you want to put your SVG files:

    AddType image/svg+xml svg
    AddType image/svg+xml svgz
    AddEncoding gzip svgz
  • Publish your SVG files into a directory on your web server. If you like, you can compress the svg file using gzip to produce an svgz file and publish the (much smaller) svgz file instead. This will reduce transmission times if that is a problem for you.
  • Insert the following tag in an HTML page whenever you want to display an SVG file:

    <object data="your.svg" width="400" height="300" type="image/svg+xml"></object>

    Don't be tempted to use the (otherwise perfectly correct) empty element syntax:

    <object ... type="image/svg+xml"/>

    ...because Internet Explorer (version 6 and maybe other versions) gets confused and stops rendering any HTML after it.
