Syntax of the simple XML Input File

A Simple Example

As an example of how simple it is to generate the XML file describing your data, the following is all that is needed to create a chart with 10 hours of CPU utilization data in it: 

<?xml version="1.0" encoding="EBCDIC-CP-US"?>
<chart>
    <title>Daily CPU Usage</title>
    <series>
        <datum y="17"/>
        <datum y="35"/>
        <datum y="86"/>
        <datum y="99"/>
        <datum y="99"/>
        <datum y="99"/>
        <datum y="99"/>
        <datum y="74"/>
        <datum y="41"/>
        <datum y="9"/>
    </series>
</chart>

A More Complex Example

A summary of the syntax is shown below in the form of an example XML input file with most options specified. If you transform this example into SVG the result is ghastly, but it illustrates where most tags go. Basically, the input file can contain:

  • One or more data series (the data points for the lines you want to plot) identified by <series> elements.

  • Optional axis information (range, title etc) identified by an <axis> element

  • Optional layout information (size of the chart, legend position etc) identified by a <layout> element

  • Optional style information (what colors and line styles to use, background images etc) identified by a <style> element

All you really need to supply is your chart data. 

Default axis, layout and style information will be applied if you don't specify any.  Axis scale and range will be computed automatically. If you don't like the default chart style, you can specify your own by supplying a <style> tag. In addition, you can override the style of individual data series by specifying a style attribute on the appropriate <series> tag. 

<?xml version="1.0" encoding="utf-8"?>
<chart debug="true">

    <title>Your Chart Title</title>

    <series>
        <title>Your First Data Series</title>
        <datum y="33"/>
        <datum y="97"/>
        <datum y="35"/>
        <datum y="23"/>
        <datum/> <!-- Missing value -->
        <datum y="81">Isolated Value</datum>
        <datum/> <!-- Missing value -->
        <datum y="70"/>
        <datum y="55"/>
    </series>

    <series style="stroke:orange;stroke-width:2;stroke-dasharray:10,5;filter:none">
        <title>Your Second Data Series</title>
        <datum y="6"/>
        <datum y="66"/>
        <datum y="13"/>
        <datum y="39"/>
        <datum y="87"/>
    </series>

Complete Syntax as defined by XML Schema

The syntax of the simple XML input file is fully described by the lineChart.xsd XML Schema Definition file.

The schema is not essential for actually creating a chart, but it is useful if you want to edit the simple XML file with a schema-aware XML editor and be prompted to input tags in the appropriate places. If you have a schema-aware XML editor such as XMLSPY 2005 Home Edition (download for free) or IntelliJ IDEA (not free, but very Very VERY good), then you should ensure that the XML input file begins with the following: 

<?xml version="1.0" encoding="utf-8"?>
<chart xmlns="http://fleabag.org/lineChart"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://fleabag.org/lineChart file:lineChart.xsd">
       .
       .
</chart>

This will enable your schema-aware editor to prompt you to input tags and attributes in the correct places. Note that before trying to transform your file into SVG you must remove the default namespace (xmlns) attribute, otherwise you will end up with an (effectively) empty output file. Optionally, you can also remove the xmlns:xsi and xsi:schemalocation attributes as these are only used when you use a schema-aware editor, or when you use a validating XML parser - neither of which are essential to the cause.

In short, before you transform the XML input file into SVG, it should look like just: 

<?xml version="1.0" encoding="utf-8"?>
<chart>
       .
       .
</chart>
