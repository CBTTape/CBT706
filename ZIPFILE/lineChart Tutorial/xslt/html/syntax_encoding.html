<?xml encoding="..."?>

Syntax

<?xml version="1.0" encoding="utf-8"?>

Synopsis

This is an optional prolog to any xml file. If you are running the XSLT processor on an IBM mainframe, then you  should specify:

<?xml version="1.0" encoding="EBCDIC-CP-US"?>

This is how you tell the XSLT processor that the characters in the XML input file are encoded in EBCDIC. The output SVG file will be in the character set defined by the <xsl:output> tag in the XSLT stylesheet (lineChart.xsl). 

Otherwise, for non-EBCDIC platforms, you should specify:

<?xml version="1.0" encoding="utf-8"?>
